# Module 4 — CDC / Real-Time Layer

**Status:** ⏳ Scheduled for M7 (5 Oct – 2 Nov 2026)
**Owner:** Abdullah + Omer

## Description

Captures database changes (INSERT, UPDATE, DELETE) in near-real-time using
Debezium and Apache Kafka, and delivers normalised change events to the data warehouse.

## KPI Target

End-to-end latency under 30 seconds from source change to warehouse delivery.

## Prerequisites

- Apache Kafka running in Docker Compose (set up in M1)
- Omer to complete Kafka self-study during M5 (August) — not October
